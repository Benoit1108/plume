# M1.1 — Cadrage : Répertoire + Login front

> Slice vertical : domaine → Doctrine → API `/api/v1` → écran Nuxt. Prérequis M0 OK.
> Réf. : [note M1](M1-conception.md), [glossaire](../GLOSSAIRE.md). Métier FR / code EN.

## 1. Objectif du slice
Constituer et consulter la **base de cibles** (Organisations + Contacts), scoped par tenant, et **se connecter** depuis le front (token + refresh). C'est la fondation que M1.2 (pipeline) réutilisera.

**Dans le périmètre** : login front, CRUD Organisation/Contact, filtres/liste, import CSV, `doNotContact` (RGPD).
**Hors périmètre** (slices suivants) : pistes, relances, IA, dashboard.

## 2. Domaine (contexte Directory)

**Agrégat `Organization`** (racine)
| Champ | Type | Notes |
|---|---|---|
| `id` | `OrganizationId` (VO, uuid) | |
| `tenantId` | `TenantId` | fixé depuis l'utilisateur courant |
| `name` | string (non vide) | |
| `type` | `OrganizationType` | `PUBLISHER \| AV_STUDIO \| AGENCY \| OTHER` |
| `website` | `?string` (URL) | |
| `country` | `?CountryCode` | ISO 3166-1 alpha-2 |
| `workingLanguages` | `LanguageCode[]` | ISO 639-1 |
| `segments` | `Segment[]` | réutilise l'enum existant |
| `notes` | `?string` | |
| `doNotContact` | bool | défaut false |
| `contacts` | `Contact[]` | **entités dans l'agrégat** |

**Entité `Contact`** (dans l'agrégat) : `id` (`ContactId`), `fullName`, `role?`, `email?` (`EmailAddress`), `phone?`, `linkedinUrl?`, `preferredLanguage?` (`LanguageCode`), `doNotContact` (bool).

**Invariants** : `name` non vide ; **email unique par organisation** ; `TenantId` cohérent ; marquer `doNotContact` sur l'org propage le blocage (futur envoi). Méthodes : `rename`, `updateProfile`, `addContact`, `updateContact`, `removeContact`, `markDoNotContact`, `contact.markDoNotContact`.

**VOs nouveaux** : `OrganizationType`, `EmailAddress` (validée), `CountryCode`, `LanguageCode`. `OrganizationId`/`ContactId` = VO uuid `Stringable` (comme `LeadId`).

## 3. Application (CQRS)
Commands : `CreateOrganization`, `UpdateOrganization`, `AddContact`, `UpdateContact`, `RemoveContact`, `MarkOrganizationDoNotContact`, `ImportOrganizationsCsv`.
Queries : `ListOrganizations(filters, page)`, `GetOrganization(id)`.
Handlers = marqueur `CommandHandler`/`QueryHandler` (pas d'attribut framework — deptrac).

## 4. Contrat d'API (API Platform, DTO + State Providers/Processors)

Base `/api/v1`, JWT requis, auto-scopé tenant, erreurs RFC 7807.

| Méthode | Route | Corps / Query | Réponse |
|---|---|---|---|
| POST | `/organizations` | `{name,type,website?,country?,workingLanguages[],segments[],notes?}` | 201 `OrganizationResource` |
| GET | `/organizations` | `?type=&segment=&language=&q=&page=&itemsPerPage=` | 200 collection paginée |
| GET | `/organizations/{id}` | — | 200 `OrganizationResource` |
| PATCH | `/organizations/{id}` | champs modifiables (dont `doNotContact`) | 200 |
| POST | `/organizations/{id}/contacts` | `{fullName,role?,email?,phone?,linkedinUrl?,preferredLanguage?}` | 201 `ContactResource` |
| PATCH | `/organizations/{id}/contacts/{contactId}` | champs | 200 |
| DELETE | `/organizations/{id}/contacts/{contactId}` | — | 204 |
| POST | `/organizations/import` | `multipart/form-data` fichier CSV | 200 `ImportReport` |

**`OrganizationResource`** (DTO exposé) : `id, name, type, website, country, workingLanguages[], segments[], notes, doNotContact, contacts: ContactResource[], createdAt`.
**`ContactResource`** : `id, fullName, role, email, phone, linkedinUrl, preferredLanguage, doNotContact`.
**`ImportReport`** : `{ createdOrganizations, updatedOrganizations, createdContacts, skippedRows: [{line, reason}] }`.

*Auth (existant)* : `POST /api/v1/login_check {email,password}` → `{token, refresh_token}` ; `POST /api/v1/token/refresh` (form `refresh_token`) → nouveau couple.

## 5. Import CSV

**Format** : UTF-8, séparateur `,`, en-tête obligatoire. **1 ligne = 1 contact** (l'organisation est dédupliquée par regroupement). Une ligne sans champ contact crée l'organisation seule.

Colonnes :
```
organization_name, organization_type, website, country, working_languages, segments,
contact_full_name, contact_role, contact_email, contact_phone, contact_linkedin, notes
```
- `working_languages`, `segments` : valeurs séparées par `;` (ex. `EN;FR`).
- `organization_type` : `PUBLISHER|AV_STUDIO|AGENCY|OTHER` (défaut `OTHER`).

**Dédoublonnage** (léger) : organisation identifiée par `name` normalisé (trim + minuscules + accents retirés) **+** domaine de `website`/`contact_email` si présent. Contact dédupliqué par `email` (ou `fullName` si pas d'email) au sein de l'org.

**Rapport** : compteurs + lignes ignorées (avec motif : type invalide, email invalide, doublon strict…).

## 7. Frontend (Nuxt + Nuxt UI)

## 12. Écarts d'implémentation (constatés à la revue de santé du 2026-07-13)

La note ci-dessus reflète l'intention initiale ; le livré diverge sur ces points, assumés :

| Prévu | Livré | Pourquoi |
|---|---|---|
| Contacts en table enfant (one-to-many) | Colonnes **JSONB** sur la racine | **ADR-0012** (domaine pur, agrégat chargé/sauvé en bloc) |
| Import en `multipart/form-data` | POST JSON `{content, delimiter}` | Plus simple des deux côtés ; borné (1 Mo / 1000 lignes) |
| Rapport `{createdOrganizations, updatedOrganizations, createdContacts, skippedRows}` | `{imported, skipped, failed, errors[]}` | Pas de mise à jour des existants en V1 : skip pur, plus prévisible |
| Dédoublonnage nom normalisé + domaine web/email | Nom seul (minuscules, trim) + **unicité en base par tenant** | Décision D1 ; la déduplication riche reviendra en M3 |
| Commande `ImportOrganizationsCsv` | Service d'Application `OrganizationImporter` | 1 ligne = 1 commande = 1 transaction (résilience) ; l'orchestration n'est pas une commande |
| Commande `MarkOrganizationDoNotContact` | Champ du PATCH (cf. micro-décision 10.2) | Et **réversible** avec events dans les deux sens (décision D3) |
| Tokens front en `localStorage` | **Cookies** `secure`/`lax` + rotation single_use + invalidation au logout | Durcissement fait dès M1.1 (revue de santé) |
| Colonne CSV `contact_linkedin` | Abandonnée | Non demandée ; réintroduction triviale si besoin |
| Filtres `segment`/`langue` + pagination | `type` + `q` + **pagination réelle** | Filtres JSONB différés (volume faible) — reste tracé en ROADMAP |
